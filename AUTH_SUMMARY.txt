3. AUTHENTICATION FLOW
================================================================================

REGISTRATION:
  1. User fills form in LoginPage (email, password, role, company)
  2. → register() in auth.js
  3. → Validate inputs, rate limit, security checks
  4. → Store in pending_registrations table
  5. → Send email with 6-digit verification code
  6. → Show ConfirmEmailPage
  7. → User enters code + password
  8. → verifyEmail() validates code
  9. → supabase.auth.signUp() creates in auth.users
  10. → Insert into public.users table
  11. → If vendor: create vendors entry
  12. → Logged in! Show Dashboard

LOGIN:
  1. User enters email + password in LoginPage
  2. → login() in auth.js
  3. → Rate limit check (5 attempts per 15 minutes)
  4. → supabase.auth.signInWithPassword()
  5. → Fetch user profile from public.users
  6. → Check if active
  7. → Detect legacy user if needed
  8. → Save to localStorage ("agro_auth")
  9. → Logged in! Show Dashboard

PASSWORD RECOVERY:
  1. User clicks "Forgot Password" in LoginPage
  2. → ForgotPasswordPage (email or phone tab)
  3. → resetPasswordByEmail() or resetPassword()
  4. → supabase.auth.resetPasswordForEmail()
  5. → User receives email link
  6. → Clicks link → ResetPasswordPage loads
  7. → updatePassword() validates session
  8. → supabase.auth.updateUser({ password })
  9. → Redirects to LoginPage

LOGOUT:
  1. User clicks logout button
  2. → signOut() hook function
  3. → supabase.auth.signOut()
  4. → clearSession() removes localStorage
  5. → setUser(null)
  6. → Show LoginPage

DELETE ACCOUNT:
  1. User clicks delete in profile page
  2. → DeleteAccountModal asks for password
  3. → deleteAccount() in auth.js
  4. → Delete from public.users (cascades)
  5. → Delete vendors if exists
  6. → supabase.auth.signOut()
  7. → Logged out, show LoginPage

4. DATABASE TABLES (AUTH-RELATED)
================================================================================

users (Main User Profile)
  id              UUID (Supabase Auth ID)
  email           TEXT UNIQUE
  email_verified  BOOLEAN
  name            TEXT
  phone           TEXT
  password_hash   TEXT (empty - uses Supabase Auth)
  profile_photo_url TEXT
  role            ENUM: admin | pivo | vendor | buyer
  city            TEXT
  notes           TEXT
  active          BOOLEAN
  created_at      TIMESTAMPTZ

vendors (Vendor Profile - linked to users)
  id              UUID
  user_id         UUID → users.id (UNIQUE)
  name            TEXT
  phone           TEXT
  photo_url       TEXT
  city            TEXT
  notes           TEXT
  created_at      TIMESTAMPTZ

pending_registrations (Temporary - stores during registration)
  id              UUID
  email           TEXT UNIQUE
  name            TEXT
  phone           TEXT
  role            ENUM
  city            TEXT
  notes           TEXT
  verification_code VARCHAR(6)
  expires_at      TIMESTAMPTZ (24 hours)
  created_at      TIMESTAMPTZ

5. USER ROLES & PERMISSIONS
================================================================================

ROLES:
  admin   - System administrator (manages users, blocks accounts)
  pivo    - Gestor/Purchase coordinator (creates campaigns, approves vendors)

  buyer   - Producer/Farmer (views campaigns, places orders - no password)
